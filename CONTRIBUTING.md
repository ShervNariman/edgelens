# Contributing to MotionGuard

Thank you for helping make frontend motion more reliable.

## Before opening a pull request

1. Create or reference an issue with observable acceptance criteria.
2. Keep one concern per pull request.
3. Add tests for behavior and failure paths.
4. Run `pnpm qa`.
5. Include a five-loop evidence section in the PR description.

## Pull request evidence

- Exact commands and environment.
- Tests added or changed.
- Accessibility and reduced-motion impact.
- Security and untrusted-input impact.
- Screenshots or recordings when behavior is visual.
- Known limitations and follow-up issues.

## Commit and merge policy

Use focused commits. Pull requests are squash-merged only after CI, QA, and senior code-health
review pass.
